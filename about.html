How does it work ?

Basics

Qem aims to be easy to use and to provide an easy and simple way of writing and using models.
To be able to do that Qem provides some C++ macros, and a set of tools to read, write and watch models.
Basically a model must be declarated with a defined set of members, each defined with a name and a metatype.
Each metatype must have been declared previously with an other macro.
You won't be able to directly access a model object, only its unique id.
You will use this id to create the following workers : Reader, Writer, Watcher.

Macros

Qem mainly uses macros to provide a easy way of writing models.
There is basically two macros in Qem :

  • QEM_MODEL_FIELD : used to declare a metatype
  • QEM_MODEL : used to declare a model
This is great for providing developer-friendly and easy to use features, but it raises a few problems :
  • Auto-completion in your IDE, some work has been done to improve that point, and it should work fine now in most IDE
  • Qt's MOC preprocessor doesn't expand macros and as it is the point of Qem it's kind of a big deal !
The solution to this is for Qem to have his own preprocessor generator.

qemPreProc

qemPreproc is a simple tool that you need to call (there is also a CMake macro to ease your pain ;) )
the header files which contains QEM macros.

This tool will generate empty class definitions with the needed signals to give MOC the expanded versions of our macros.
You can call the tool manually :
qcePreProc [-o<outputFile>] <inputFile>
